# 弋阳县/万年县充换电设施规划系统 - 动态架构

## 架构说明

采用前后端分离的动态架构：

```
┌─────────────┐      HTTP/JSON      ┌──────────────┐
│   浏览器     │ ←─────────────────→ │  Flask 后端   │
│  (前端)      │    RESTful API      │   (app.py)   │
└─────────────┘                      └──────────────┘
                                            ↓
                                     ┌──────────────┐
                                     │   SQLite DB  │
                                     │ yiyang_ev.db │
                                     └──────────────┘
```

## API 端点

| 端点 | 说明 | 返回数据 |
|------|------|----------|
| `GET /` | 主页面 | HTML |
| `GET /api/config` | 地图配置 | 天地图Key、中心点、样式配置 |
| `GET /api/boundary` | 县域边界 | 111个坐标点 |
| `GET /api/stations/urban` | 现有充电站 | 16座，含精确坐标 |
| `GET /api/stations/gas` | 加油站 | 26座 |
| `GET /api/stations/planned` | 规划充电站 | 34站 |
| `GET /api/stats` | 统计摘要 | 汇总数据 |
| `GET /api/economic` | 经济数据 | 2016-2025年时间序列 |

## 前端功能

- **动态数据加载**：页面启动时通过 Fetch API 并行加载所有数据
- **天地图集成**：使用天地图 API v4.0，内置底层 CSS 拦截去字注记，提供更洁净的底布
- **高精度县域聚光灯+3D悬浮**：采用百万级坐标高精度 GeoJSON 两县边界，支持内外遮罩反转和两县合并联查底图开窗，彻底解决多图层嵌套覆压问题，提供类似规划图纸的阴影效果及立体悬浮感
- **多县域无缝联动**：完美支持万年县、弋阳县单县无缝切换以及两县一体化全景展示，解决并修正了异步数据加载引起的死循环重绘闪烁问题，且联查时默认展示洁净行政区块版图模式
- **乡镇行政区划精确定位**：通过 Python `shapely` 库对 GeoJSON 面几何源数据进行了中心质心(Centroid)真实经纬度坐标精确运算纠偏，解决了乡镇名称标注偏移堆砌的问题，并在底图上支持叠图查看两县各乡镇辖权网格边界
- **底图自适应深浅色**：可在（地形图/卫星影像/标准矢量）三种底版间自由切换，程序会自动调节县域内外的色彩压暗、边界光晕粗细及其描边颜色
- **多层级规划图例**：通过不同色彩、形状、层级光晕将点位按：现有充电站、全域规划点、专用点位、景区点位、加油站等划分，具备极高的图例识别度
- **所见即所得高级导出器**：内置 `html2canvas` 高清水印图片截取引擎（已支持截取规划正方形排版框+规范标题），可分离导出合成图、光板底图以及纯透明底点位图（此功能可在HTML中配置显隐）
- **5种显示模式系统**：支持侧边栏一键切换显示图层组合及独立开关控制
- **交互式浮窗**：点击标注实时提取气泡数据信息

## 优势

1. **前后端分离**：便于维护和扩展
2. **数据实时性**：修改数据库后刷新页面即可看到更新
3. **API 可复用**：其他应用可直接调用 API
4. **易于部署**：可部署到云服务器供多人访问
5. **可扩展性**：易于添加新功能（搜索、筛选、统计图表等）

## 技术栈

- **后端**：Flask 3.1.3 + Flask-CORS 6.0.2
- **数据库**：SQLite 3
- **前端**：原生 HTML/CSS/JavaScript（无框架依赖）
- **地图**：天地图 API v4.0
- **数据处理**：Pandas 3.0.1, Shapely 2.0.7

## 开发模式

Flask 运行在 debug 模式，代码修改后自动重启。生产环境需要：

```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

配合 Nginx 反向代理使用。
